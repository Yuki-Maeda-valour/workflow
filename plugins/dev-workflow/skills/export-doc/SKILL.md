# export-doc — クライアント向けドキュメント出力

doc/ の文書(要件定義・設計・計画など)を、クライアントに渡せる形式(PDF / xlsx / HTML)へ変換する。中身の鮮度は /update-doc・/reflect-decisions が担い、本 skill は**見せられる形にすること**だけを担う。

## 原則

1. **正本は doc/、出力は派生物**。doc/ は読み取り専用で 1 文字も変更しない。書き込むのは出力先(既定 `export/`)のみ。出力物を直接手直ししない(直したくなったら doc/ を直して再出力する)
2. **サニタイズゲート**。クライアント向け出力では、内部専用情報(引き継ぎ・地雷、管理タグ、未定項目、AI 運用向け記述)の扱いを**必ず確認してから**出力する(`--yes` でも省略しない)。社内向け(`--internal`)のみ省略可
3. **機密検査**。profile の `secret_paths` にあるファイルの値・キー名や秘密らしき文字列が出力に混入していないか、生成前後で検査する
4. **ツールをハードコードしない**。変換ツールは実行時に検出する。初回ダウンロードやインストールを伴う手段は勝手に実行せず、確認してから。何も使えなければ HTML 出力に縮退し、手順(ブラウザ印刷)を案内する
5. **冪等**。同じ入力・同じ設定なら同じ出力になる。出力ファイル名は「文書名_YYYY-MM-DD」で揃える

## 引数

| 引数 | 内容 |
|---|---|
| 対象文書(複数可) | doc/ 配下のファイル名・番号(例: `03` `07`)。`--all` で索引の全文書 |
| `--pdf` / `--xlsx` / `--html` | 出力形式(複数指定可。省略時は Phase 1 で確認) |
| `--internal` | 社内向け(サニタイズ確認を省略。機密検査は行う) |
| `--merged` | PDF を 1 ファイルに結合した版も出力 |
| `--out=<dir>` | 出力先(既定: `export/YYYY-MM-DD/`) |
| `--yes` | サニタイズ以外の確認(形式・ツール採用)をスキップ |

## Phase 1: 対象・形式・用途の確認

1. `.claude/project-profile.yml` があれば Read し、`root` / `secret_paths` を解決する(無くても続行)。doc ディレクトリを検出する(`doc/` → `docs/`)
2. doc/README.md(索引)から文書一覧と状態を把握する。**状態が 🔴 未確定・🟡 検討中の文書を出力対象に含む場合は、その旨を注意として添える**(古い・未確定のまま提出する事故の防止。/update-doc・/reflect-decisions で最新化してからの出力を推奨)
3. 引数で決まっていない事項を **AskUserQuestion 1 回**で確認する: 対象文書(全部 / 選択)・形式(PDF / xlsx / 両方)・用途(クライアント提出 = サニタイズあり / 社内 = なし)
4. xlsx の主対象は**表を持つ文書**(03 要件一覧・07 見積もり/マイルストーン・索引)。表の無い文書は xlsx 対象外である旨を伝える(PDF / HTML を案内)

## Phase 2: 変換環境の検出

利用可能なツールを検出し、採用する経路を提示する(縮退したときは必ず報告に明記):

| 用途 | 検出順 | すべて無い場合の縮退 |
|---|---|---|
| Mermaid 図の画像化 | `mmdc`(mermaid-cli)→ `npx -y @mermaid-js/mermaid-cli`(要 Node。初回 DL は確認) | 図を出力から外し「図は原本参照」と注記+報告 |
| PDF 生成 | Chrome / Chromium / Edge のヘッドレス印刷(`--headless --print-to-pdf`)→ `pandoc`(日本語対応エンジンがある場合)→ `soffice --convert-to pdf` | HTML のみ出力し、ブラウザで開いて印刷([⌘+P] → PDF 保存)する手順を案内 |
| xlsx 生成 | `python3` + `openpyxl` → `pip install openpyxl` を提案(確認) | スクリプトが CSV に自動縮退(Excel で開ける。文字コードは BOM 付き UTF-8) |

- 検出はコマンドの存在確認(`command -v` 等)で行い、バージョン・パスを決め打ちしない
- ヘッドレス印刷は OS ごとに実行パスが異なるため、実在を確認したパスを使う(macOS はアプリバンドル内、Linux は `google-chrome` / `chromium` 等)

## Phase 3: サニタイズ(クライアント向けのみ)

**doc/ は変更せず**、scratchpad に作業コピーを作って適用する。除外・変換の候補を一覧で提示し、確認を取ってから実施する:

| 項目 | 既定の扱い |
|---|---|
| 05 の「引き継ぎ・地雷」節 | **除外**(内部知見のため) |
| 管理タグ `[実][決][将][前]` | 日本語へ置換(実装済み / 決定済み / 将来対応 / 前提)。凡例ごと残す選択も可 |
| 「（未定）」・ガイド文「（〜を記述）」 | 「未定」表記に整えて残す(隠さない — 未定を隠して提出すると合意形成を壊す)。行ごと除外したい場合は個別指定 |
| 暫定マーク「暫定(日付)」 | 残す(クライアントと共有すべき状態情報) |
| 環境変数のキー名一覧(05) | 確認して判断(構成情報として見せる場合もある) |
| doc 間の相対リンク | PDF ではリンク切れになるため「→ ○○(別紙)」等の文書名表記へ変換 |
| AI 運用向け記述(profile・skill への言及) | 除外 |

- **機密検査(事前)**: 作業コピーに対し、`secret_paths` のキー名・秘密らしき長い英数文字列(トークン様)・実在の認証情報パターンを Grep し、検出したら該当箇所を提示して指示を仰ぐ

## Phase 4: 変換実行

1. **Mermaid の事前レンダリング**: 作業コピー内の ```mermaid ブロックを SVG(不可なら PNG)に変換し、画像参照へ差し替える
2. **HTML 生成**: `pandoc` があれば `templates/print.css`(このスキルと同階層)を適用して変換。無ければ実行者自身が print.css を内蔵した自己完結 HTML を 1 文書ずつ生成する(外部 CDN に依存しない・画像はローカル参照または埋め込み)
3. **PDF 生成**: Phase 2 で採用した経路で HTML → PDF(A4 縦・余白は print.css に従う)。`--merged` 指定時は結合版も作る(結合ツールが無ければ個別 PDF のみ+その旨報告)
4. **xlsx 生成**: `python3 {このスキルの}scripts/md_tables_to_xlsx.py <対象md...> -o <出力>.xlsx` を実行する(1 表 1 シート、シート名は「文書名 + 見出し」。openpyxl が無ければ CSV に自動縮退)
5. **表紙(任意・PDF のみ)**: クライアント向けでは「タイトル・プロジェクト名・日付・版(索引の最終更新日)」の表紙ページを先頭に付けられる(Phase 1 で確認)
6. 出力先: `--out` または既定 `export/YYYY-MM-DD/`。同名ファイルは上書き(冪等)

## Phase 5: 検証と報告

1. **出力の実在確認**: 各ファイルの存在・サイズ・(可能なら)PDF のページ数・xlsx のシート数を確認する。0 バイト・変換失敗はエラーとして再試行または縮退
2. **機密検査(事後)**: テキスト抽出できる出力(HTML / CSV / xlsx のセル)に対して Phase 3 と同じ検査を再実行する
3. **文字化けチェック**: 生成 HTML の日本語フォント指定(print.css のフォントスタック)を確認し、PDF は最低 1 ファイルをユーザーが開いて確認するよう促す(豆腐・欠落は環境フォント起因のため機械検出しきれない)
4. **gitignore の提案**: 出力先が git 管理下で `.gitignore` に無ければ追記を提案する(提出物の誤コミット防止。成果物も版管理する方針ならスキップでよい)
5. **報告**: 出力一覧(形式・ページ/シート数)/ 採用した変換経路(縮退した場合はその内容)/ サニタイズで除外・変換した項目 / スキップした図 / 再生成の方法(同じ引数で再実行すれば同じ出力になる旨)

## 最終ゲート(完了報告前セルフチェック)

- [ ] doc/ 配下を 1 文字も変更していない(書き込みは出力先と scratchpad のみ)
- [ ] クライアント向け出力でサニタイズ確認を省略していない(`--internal` 時を除く)
- [ ] 機密検査を事前・事後の両方で実施し、検出ゼロを確認した
- [ ] 縮退・スキップ(図・結合・xlsx→CSV)を黙って行っていない(すべて報告に明記した)
- [ ] 未確定(🔴/🟡)の文書を出力に含めた場合、その旨を注意として伝えた
- [ ] 出力物を直接手直ししていない(修正は doc/ → 再出力の導線を案内した)
- [ ] 重いダウンロード・インストールを確認なしで実行していない
